# Fit/Gap-анализ

**Список систем:**

* ИС «Алмазы» (существующая) — отбор алмазов для цифрового аукциона (новая функциональность) — необходимо добавить модуль/раздел для отбора камней под онлайн-аукцион, присвоения специальных идентификаторов, подготовки ведомостей и планирования работ.
* ИС «Алмазы» (существующая) — отслеживание статуса подготовки алмазов к онлайн-аукциону (новая) — требуется интеграция статусов оцифровки и готовности к публикации.
* ИС «Логистика» (существующая) — этапы отправки/возврата алмазов в лабораторию (новая функциональность) — отсутствуют специализированные этапы и статусы для перевозки в лабораторию и обратно.
* ИС «Логистика» (существующая) — отслеживание специальной защищённой перевозки ювелирных алмазов (существующая) — требуется доработка под новые маршруты (офис → лаборатория → офис → покупатель) и усиление контроля.
* ИС «Аукцион» (существующая) — загрузка и отображение аналитики/оцифрованных данных по алмазам (новая функциональность) — полностью отсутствует возможность загружать и показывать покупателям результаты оцифровки (3D-модели, спектры, включения и т.д.).
* ИС «Аукцион» (существующая) — проведение онлайн-аукциона (новая/расширенная) — требуется доработка для публикации цифровых лотов, онлайн-ставок и работы с цифровыми данными вместо физического просмотра.
* ИС «Бухгалтерия» (существующая) — актирование и оплата услуг лаборатории (новая) — необходимо добавить процессы работы с новым подрядчиком (ИС «Лаборатория»).
* ИС «Бухгалтерия» (существующая) — ведение контрактов и актирование услуг перевозчиков (существующая) — требуется расширение под новые виды перевозок (в лабораторию).
* ИС «Лаборатория» (новая) — оцифровка алмазов и передача результатов (новая система) — полностью новая система у внешнего подрядчика, требуется интеграция.
Облачные сервисы (новые/внешние) — публикация данных для клиентов (новая) — в промежуточном решении нужен частный облачный сервис; в целевом — перенос на собственные серверы.

**Список интеграций:**

* ИС «Алмазы» → ИС «Логистика» — планирование и перевозка алмазов (существующая) — автоматически — существующие данные о партиях алмазов + новые данные по отобранным для онлайн-аукциона камням.
* ИС «Логистика» → ИС «Алмазы» — возврат алмазов из лаборатории с обновлённым статусом (новая) — автоматически (целевое) / вручную (промежуточное) — статусы оцифровки и возврата камней.
* ИС «Лаборатория» → ИС «Аукцион» — передача описания и аналитики оцифрованных алмазов (новая) — автоматически (целевое) / через Excel + ручная загрузка (промежуточное) — полные цифровые данные (3D, спектры, включения, оценки и т.д.).
* ИС «Лаборатория» → ИС «Бухгалтерия» — актирование и оплата услуг оцифровки (новая) — автоматически (целевое) / вручную (промежуточное) — данные об оказанных услугах, объёмах, стоимости.
* ИС «Логистика» → ИС «Бухгалтерия» — актирование и оплата услуг транспортной компании (существующая) — автоматически — данные о выполненных перевозках (включая новые маршруты в лабораторию).
* ИС «Алмазы» / ИС «Аукцион» → Облачный сервис — публикация данных для покупателей (новая) — автоматически (целевое) / вручную через Excel (промежуточное) — цифровые карточки алмазов и аналитика.
* ИС «Аукцион» → ИС «Логистика» — передача данных победителям для отправки камней (новая) — автоматически (целевое) / вручную (промежуточное) — информация о проданных лотах и покупателях.

**Ключевые пробелы, требующие внимания:**

* Промежуточное решение сильно зависит от Excel и ручных операций → высокие риски ошибок и задержек.
* Отсутствие единого отслеживания статуса всего процесса (от отбора до отправки покупателю).
* Конфиденциальность и безопасность передачи данных оцифровки и коммерческой информации.
* Зависимость от внешнего облака и подрядчика-лаборатории в первые месяцы.
